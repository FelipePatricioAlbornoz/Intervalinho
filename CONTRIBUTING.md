# Contribuindo para o Intervalinho

Antes de mais nada, obrigado por considerar contribuir com o Intervalinho! São pessoas como você que tornam o código aberto incrível.

## Como posso contribuir?

Existem muitas maneiras de contribuir, desde escrever tutoriais ou posts para o blog, melhorar a documentação, enviar relatórios de bugs e solicitações de recursos ou escrever código que pode ser incorporado ao próprio Intervalinho.

### Reportando Bugs

- **Certifique-se de que o bug ainda não foi relatado** pesquisando no GitHub em "Issues".

- Se você não encontrar uma issue aberta que aborde o problema, abra uma nova. Certifique-se de incluir um **título e descrição clara**, o máximo de informações relevantes possível e um **exemplo de código** ou um **caso de teste executável** demonstrando o comportamento esperado que não está ocorrendo.

### Sugerindo Melhorias

- Abra uma nova issue com a etiqueta "enhancement".

- Descreva claramente a melhoria e a motivação para ela.

## Primeiros Passos

1. Faça um fork do repositório.

2. Clone seu fork: `git clone https://github.com/seu-nome-de-usuário/intervalinho.git`
3. Navegue até o diretório do projeto: `cd intervalinho`
4. Instale as dependências: `npm install`
5. Crie uma nova branch para sua funcionalidade ou correção de bug: `git checkout -b minha-nova-funcionalidade`

## Estilo de Código

Por favor, tente seguir o estilo de codificação dos arquivos existentes. Isso ajuda a manter a base de código consistente e fácil de ler.

## Envio de Alterações

1. Confirme suas alterações com uma mensagem de commit clara e descritiva.

2. Envie sua branch para o seu fork: `git push origin minha-nova-funcionalidade`
3. Abra um pull request para a branch `main` do repositório original.

4. Certifique-se de que a descrição do PR descreva claramente o problema e a solução. Inclua o número da issue relevante, se aplicável.

Agradecemos novamente sua contribuição!
